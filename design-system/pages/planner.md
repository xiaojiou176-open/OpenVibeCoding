# Planner Override

This page is the **planning control desk**.

## Primary job

Make the operator understand, in one glance:

1. which wave deserves attention first
2. what is missing from that wave
3. which surface should open next

## Rules

- The first viewport must always surface one **priority wave** instead of four equal actions.
- Metrics are supporting instrumentation, not the hero.
- `worker contracts`, `unblock tasks`, and `wake policy` must read like planning posture, not like raw artifact counts.
- Empty state should still feel operational: show the next three moves, not a blank warehouse.
- Inspection archive is second-layer only; it may deepen the planning picture, but it must never repeat the triage row one-to-one.

## Avoid

- table-first planner layout
- action menus that make every destination feel equally important
- artifact-gallery empty states
- duplicated row metadata across first and second layer
