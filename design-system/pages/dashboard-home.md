# Dashboard Home Override

Use the home page as a **briefing deck**, not a documentation index.

## Primary job

Teach the product in under 20 seconds:

- what OpenVibeCoding is
- why it exists
- what the operator loop is
- where to start

## Rules

- Hero must lead with the command-tower sentence and the pain hook.
- The next visual layer must be the operator loop: plan / delegate / track /
  resume / prove.
- The philosophy layer (`Prompt / Context / Harness`) belongs on the page, but
  below the core loop.
- Adoption/router sections are important, but they are not the hero.
- Do not let the page feel like a catalog of links.

## Avoid

- testimonial-driven landing tropes
- multiple competing “start here” stories
- hiding the main control surfaces below too much explanation
